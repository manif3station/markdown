# DD-386

## Title

Restore the coverage, documentation, and release gates for the pdf-to-docx /
pdf-to-markdown bugfix shipped in commit `61c8ed9`.

## Method

TDD

## Problem

Commit `61c8ed9` fixed real production bugs (LibreOffice structurally cannot
export a PDF import to DOCX; CAM::PDF returns empty page text when marked
content interleaves with the graphics stack; subsetted fonts decode to byte
noise without their /ToUnicode CMaps; Tm-positioned PDFs collapse to one
line) but bypassed the gated cycle. The coverage gate was broken at `93.4%`
statement / `95.0%` subroutine for `lib/Markdown/Runner.pm`, `README.md`
still documented the removed LibreOffice pdf-to-docx backends, `Changes` and
`.env` `VERSION` were not updated, and no ticket or backlog record existed.

## Acceptance

- new tests cover every statement and subroutine the bugfix left unexercised:
  the `'`/`"`/`Td`/`TD`/`T*` operator dispatch and handlers, TJ kerning
  guards, the Tm paragraph-gap break, the surrogate-pair and array-form
  bfrange ToUnicode paths, the broken-ToUnicode fallback, the unreadable-PDF
  die, the DOCX code-block and unknown-block branches, and the LibreOffice
  scratch-dir rename into the requested target path
- the ToUnicode decode path is proven against a real hand-built PDF parsed by
  the real `CAM::PDF`, not only against mocked content trees
- Docker tests pass and coverage is `100.0%` statement and subroutine for
  `lib/Markdown/CLI.pm`, `lib/Markdown/Enhancer.pm`, and
  `lib/Markdown/Runner.pm`
- `README.md` no longer claims LibreOffice performs pdf-to-docx and documents
  the pure-Perl `CAM::PDF` + `Archive::Zip` route and the `Archive::Zip`
  dependency
- `Changes` records the bugfix and this retroactive gating as `0.15`, and
  `.env` carries `VERSION=0.15`
- cross-platform verification evidence is captured in `tickets/TESTING.md`
  (Linux Docker suite; macOS and Windows through the QEMU `macdev`/`windev`
  labs when they boot, with the outcome recorded either way)
- `cover_db` is cleaned from the skill folder
- release commit and push complete from the `markdown` repo
